# Roadmap

## Phase 1: Foundation

- FastAPI backend
- `POST /analyze-project`
- `GET /health`
- structured request and response models
- local Docker support

## Phase 2: Clarification engine

- detect project intent
- generate targeted questions
- avoid unsafe assumptions
- return structured clarification payloads

## Phase 3: Engineering pipeline

- requirement extraction
- architecture proposal
- epics generation
- security analysis
- roadmap generation

## Phase 4: Product-grade UX

- React or Next.js frontend
- nice result visualization
- example inputs and outputs
- better user flow for clarification and iteration

## Phase 5: LLM integration

- OpenAI API adapter
- Groq API adapter
- structured outputs enforcement
- prompt versioning

## Phase 6: Multi-agent evolution

- split orchestration into specialized agents
- add stronger QA and review steps
- keep the single orchestrator as the fallback contract if needed
